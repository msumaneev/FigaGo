# Менеджер запросов 1С

Навык для выполнения произвольных запросов к рабочей базе 1С и их предварительной валидации.

## Инструменты

### Выполнение запроса (Execute)

Если пользователь просит «выполни запрос», «выбери данные», «сколько записей в X»:

```bash
python .agent/skills/1c-query-manager/scripts/query_runner.py --action execute --query "ВЫБРАТЬ ПЕРВЫЕ 10 * ИЗ Справочник.Номенклатура"
```

### Выполнение из файла (Execute from file)

**Используйте `--query-file` если запрос содержит спецсимволы** (кавычки `""`, слэши `/`, номера документов типа `01/061890` и т.п.) — это решает все проблемы экранирования в PowerShell:

```bash
# 1. Записать запрос в файл (UTF-8)
# 2. Выполнить:
python .agent/skills/1c-query-manager/scripts/query_runner.py --action execute --query-file /tmp/query.txt
```

Пример содержимого `/tmp/query.txt`:
```sql
ВЫБРАТЬ Р.Номер, Р.Дата
ИЗ Документ.РеализацияТоваровУслуг КАК Р
ГДЕ Р.Номер = "01/061890"
  И Р.Дата >= ДАТАВРЕМЯ(2025, 3, 19)
  И Р.Дата < ДАТАВРЕМЯ(2025, 3, 20)
```

> **ПРАВИЛО:** Если в запросе есть строковые литералы с `/`, `"` или иными спецсимволами — ВСЕГДА используйте `--query-file`. Не пытайтесь экранировать вручную.

## ⛔ ОБЯЗАТЕЛЬНЫЕ ПРАВИЛА ЗАПРОСОВ

### 1. ЗАПРЕТ запросов по документам без ограничения даты

**КАТЕГОРИЧЕСКИ ЗАПРЕЩЕНО** выполнять запросы к таблицам `Документ.*` без фильтра по дате.
Это вызывает Full Table Scan и может повесить рабочую базу.

**ОБЯЗАТЕЛЬНО** добавляйте `И Дата > ДАТАВРЕМЯ(...)` с ограничением **не более 10 месяцев назад** от текущей даты.

✅ Правильно:
```sql
ВЫБРАТЬ ... ИЗ Документ.РеализацияТоваровУслуг КАК Р
ГДЕ Р.Дата > ДАТАВРЕМЯ(2025, 3, 1)          -- ограничение обязательно
  И Р.Номер ПОДОБНО "%061890%"
```

❌ ЗАПРЕЩЕНО:
```sql
ВЫБРАТЬ ... ИЗ Документ.РеализацияТоваровУслуг КАК Р
ГДЕ Р.Номер ПОДОБНО "%061890%"              -- НЕТ фильтра по дате!
```

> Скрипт **автоматически отклонит** запрос к `Документ.*` без `ДАТАВРЕМЯ` / `Дата` в условии `ГДЕ`.

### 2. Запрет Full Scan по ПОДОБНО

**ЗАПРЕЩЕНО** использовать `ПОДОБНО '%...%'` сразу к полям реальных таблиц (это вызывает Full Scan).
СНАЧАЛА отфильтруйте данные по индексам (дата, склад, контрагент), сохраните во временную таблицу, и уже из неё делайте поиск по подстроке.

### Валидация запроса (Validate)

Используйте перед выполнением сложных запросов, чтобы избежать таймаутов из-за синтаксических ошибок:

```bash
python .agent/skills/1c-query-manager/scripts/query_runner.py --action validate --query "ВЫБРАТЬ ..."
```

## Технические детали

- **Конфигурация** — `.agent/skills/1c-query-manager/config.json`
- Скрипт импортирует `plugin_transport.py` из корня проекта.
- Результаты возвращаются в формате JSON.
