# Инструкция по развертыванию инфраструктуры OpenSpec в Antigravity

Данная инструкция (ТЗ) описывает процесс переноса и интеграции спецификаций OpenSpec (`/openspec`) и рабочих процессов AI-разработки на другой компьютер для произвольного проекта, разрабатываемого с помощью AI-агента Antigravity.

## 📌 1. Предварительные требования

Перед началом интеграции на новом компьютере должны быть:
1.  **Среда выполнения:** Установлен `Node.js` (рекомендуется LTS, например v18 или v20) и пакетный менеджер `npm`.
2.  **IDE:** Установлен плагин или полноценное окружение `Antigravity`.
3.  **Система контроля версий (Git):** Для синхронизации файлов между компьютерами (рекомендуется).

## 📂 2. Синхронизация структуры через GitLab

Чтобы Antigravity на новом компьютере понимал ваши бизнес-процессы (создание proposal, tasks, design и т.д.), необходимо иметь конфигурационные папки целевого проекта. Поскольку используется GitLab, процесс выглядит следующим образом:

1.  **Склонируйте репозиторий проекта из GitLab:**
    ```bash
    git clone https://gitlab.com/<ваш-домен-или-группа>/<имя_проекта>.git
    cd <имя_проекта>
    ```
    Убедитесь, что в склонированном репозитории присутствуют скрытая папка `.agent/` и папка `openspec/`. *(Примечание: для новых проектов вы можете создать в GitLab шаблонный репозиторий "AI-Template", содержащий базовые настройки агента, и инициализировать форки от него).*

2.  **Структура, переносимая через GitLab:**
    *   Папка **`.agent/`** — содержит `skills/` (навыки) и `workflows/` (сценарии). Antigravity нативно считывает их и расширяет функционал (например, включает `/opsx-propose`).
    *   Папка **`openspec/`** — содержит файл конфигурации `config.yaml` с правилами проекта. Меняйте этот файл под реалии текущего репозитория, делайте коммит и отправляйте изменения (`git push`) обратно в GitLab для остальной команды.


## 🛠️ 3. Установка OpenSpec CLI

Глобально установите утилиту OpenSpec. Она необходима для того, чтобы Antigravity (с помощью скиллов) мог локально генерировать файлы спецификаций, проставлять зависимости между задачами и проверять статусы готовности фич.

В терминале на новом компьютере выполните:
```bash
npm install -g @fission-ai/openspec
```

*Для проверки корректности установки:*
```bash
openspec --version
```
Убедитесь, что команда возвращает версию утилиты без ошибок.

## 🧠 4. Интеграция и запуск пайплайна в Antigravity

Агент Antigravity нативно работает со структурой `.agent`. Дополнительная настройка правил (как в других редакторах) не требуется.

1. Откройте ваш целевой проект в IDE, где запущен Antigravity.
2. Убедитесь, что папки `.agent/` и `openspec/` находятся в корне (на одном уровне с исходным кодом).
3. При необходимости поменяйте системные переменные и настройки стека в `openspec/config.yaml`.
4. В чат агенту Antigravity запустите базовую команду для тестирования рабочего процесса:
   ```text
   /opsx-explore
   ```

Antigravity увидит команду из `.agent/workflows`, обратится к установленному глобально `openspec` CLI и начнет интерактивное взаимодействие по формированию ТЗ или ресерчу новой функциональности.

---
**Итог:** Конфигурация является переносимой модульной песочницей. Основой служит папка `.agent` (которую "читает" Antigravity) и локально установленная CLI утилита `openspec`. Как только эти компоненты добавлены в любой проект, Antigravity автоматически переводит разработку на рельсы OpenSpec.
