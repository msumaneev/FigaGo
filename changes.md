Подсистема учета и прогнозирования расхода батареи
1. Конфигурация типов индикаторов (Профили пультов)Структура: Неизменяемый класс или Enum, содержащий массивы базовых весов (долей емкости) для каждого типа пульта.Индексация: Массивы сортируются сверху вниз (индекс 0 — самая верхняя/первая тухнущая лампа, индекс N — нижняя/красная).Массивы весов (в десятичных долях):Type_3: [0.45, 0.35, 0.20]Type_4: [0.35, 0.30, 0.20, 0.15]Type_5: [0.30, 0.20, 0.20, 0.15, 0.15]Type_6: [0.25, 0.20, 0.15, 0.15, 0.15, 0.10]Type_8: [0.20, 0.15, 0.15, 0.12, 0.12, 0.12, 0.09, 0.05]Type_10: [0.13, 0.13, 0.13, 0.10, 0.10, 0.10, 0.10, 0.07, 0.07, 0.07]2. Архитектура хранения данныхТребование: Использование локальной реляционной БД (например, Room/SQLite).Таблица LampStatistics (История пробегов):id (Primary Key)lamp_index (Int) — индекс лампы.actual_distance (Float) — пройденное расстояние на этой лампе в километрах.timestamp (Long) — временная метка записи.Правило "Скользящего окна" (учет износа): * Для каждого lamp_index хранится жесткий лимит записей (константа HISTORY_LIMIT = 5 или 10).При вставке новой записи проверяется количество строк для данного lamp_index. Если их больше HISTORY_LIMIT, строка с минимальным timestamp удаляется.3. Алгоритмы и бизнес-логикаЭтап А: Расчет при отсутствии истории (Холодный старт)Входные данные: Заявленный пользователем общий пробег $D_{total}$ и выбранный профиль пульта.Расчет дистанции лампы: $E_{i} = D_{total} \times W_{i}$ (где $W_{i}$ — вес лампы из конфигурации).Остаток хода: Сумма $E_{i}$ для всех еще горящих ламп.Этап Б: Фиксация факта (Запись в БД)Триггер: UI-событие или голосовая команда "Потухла лампа".Действие: Вычислить дельту дистанции по GPS-треку (расстояние с момента старта или с момента потухания предыдущей лампы).Запись: Сохранить дельту в LampStatistics для текущего lamp_index.Этап В: Расчет прогноза (Горячий старт)Вычисление прогноза конкретной лампы ($E_{i}$):Если в LampStatistics есть записи для lamp_index: вычислить среднее арифметическое (или медиану) из поля actual_distance.Если записей нет (нижние лампы еще не тухли ни в одной поездке): использовать базовую формулу пропорции от оставшегося "теоретического" пробега.Вычисление оставшегося запаса хода ($D_{remaining}$):$$D_{remaining} = \sum_{j=m}^{last} E_{j}$$(Где $m$ — индекс верхней из оставшихся горящих ламп, $E_{j}$ — статистический прогноз для каждой лампы).Этап Г: Обработка пропусков (Edge Case)Сценарий: Пользователь пропустил фиксацию 1-й лампы и зафиксировал только отключение 2-й.Решение: Общая пройденная дистанция распределяется между пропущенными лампами пропорционально их базовым весам $W_{i}$, после чего в БД записываются два раздельных факта.
