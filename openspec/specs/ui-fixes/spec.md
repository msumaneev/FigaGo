## ADDED Requirements

### Requirement: Save Button Fix
Приложение SHALL корректно обрабатывать нажатие на кнопку "Сохранить" и вызывать соответствующую логику сохранения.

#### Scenario: Save action is triggered
- **WHEN** пользователь нажимает кнопку "Сохранить" (например, в настройках)
- **THEN** данные немедленно сохраняются, и UI реагирует корректно (возврат назад или сообщение об успехе)

### Requirement: Pause Button Activation
Приложение SHALL активировать кнопку "Пауза" после начала записи маршрута.

#### Scenario: Pause is available during tracking
- **WHEN** пользователь начал запись трека (нажал "Старт")
- **THEN** кнопка "Пауза" становится доступной для нажатия, позволяя приостановить запись


## MODIFIED Requirements

### Requirement: Save Button Fix
Приложение SHALL корректно обрабатывать нажатие на кнопку "Сохранить" и вызывать соответствующую логику сохранения без принудительной навигации.

#### Scenario: Save action is triggered
- **WHEN** пользователь нажимает кнопку "Сохранить" (например, в настройках)
- **THEN** данные немедленно сохраняются, UI реагирует корректно (сбрасывается isDirty, скрывается кнопка изменения), и не происходит возврата назад

## ADDED Requirements

### Requirement: Foreground Service Notification Sync
Уведомление в системной шторке SHALL визуально отражать текущее состояние сервиса.

#### Scenario: Updating notification action button
- **WHEN** пользователь нажимает кнопку "Пауза" или "Старт" в уведомлении
- **THEN** сервис применяет конфигурацию и вызывает обновление уведомления с новой кнопкой статуса
