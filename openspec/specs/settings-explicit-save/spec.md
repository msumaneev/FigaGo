

## ADDED Requirements

### Requirement: Явное сохранение настроек
Настройки больше не должны сохраняться в DataStore автоматически при каждом вводе пользователя. Должна быть возможность отменить или явно применить изменения.

#### Scenario: Успешное сохранение через кнопку
- **WHEN** пользователь изменяет значения полей на экране настроек, а затем нажимает кнопку "Сохранить"
- **THEN** обновленные значения переносятся из `SettingsViewModel` в `DataStore` (через `SettingsRepository`), а внутренний стейт `originalState` синхронизируется с текущим. Экран закрывается или выводит сообщение.

#### Scenario: Нажатие кнопки Назад при наличии изменений
- **WHEN** пользователь изменяет хотя бы одно поле (состояние становится "грязным") и нажимает аппаратную кнопку Назад или системную стрелку Назад
- **THEN** появляется диалоговое окно "У вас есть несохраненные изменения" с вариантами диспетчеризации: "Сохранить", "Выйти без сохранения", "Отмена".

#### Scenario: Нажатие кнопки Назад без изменений
- **WHEN** пользователь не меняет поля (или возвращает к исходным) и нажимает кнопку Назад
- **THEN** экран настроек закрывается штатно без дополнительных диалогов.

### Requirement: Выбор целых чисел через Барабан (WheelPicker)
Ввод целых значений (минуты интервалов, GPS-секунды, авто-пауза) должен происходить через графический интерфейс барабана (WheelPicker) без использования виртуальной клавиатуры и ползунков. Диапазон ограничен (напр. от 1 до 99).

#### Scenario: Использование барабана прокрутки (Wheel Picker) для выбора чисел
- **WHEN** пользователь хочет изменить целочисленный параметр (например, "Интервал мин")
- **THEN** он прокручивает вертикальный графический барабан, выбирая нужное целое число. Клавиатурный ввод полностью отсутствует, что надежно защищает от ввода некорректных знаков и ошибок конвертации '.0'.
