# Дизайн: Внедрение Динамических Цветов и Локализации (Многоязычность)

## 1. Динамические Цвета
* **`Theme.kt`**: В функции `FigaGoTheme` проверяем флаг `dynamicColor` и возможность поддержки платформой `Build.VERSION.SDK_INT >= Build.VERSION_CODES.S`.
* **Jetpack Compose**: Точечный перенос компонентов `Color.White` и `Color.Black` на токены `MaterialTheme.colorScheme.surface`, `onSurface`, `surfaceVariant` и т.д. Оставляем специфичные цвета для батареи и карт в виде захардкоженных констант для безопасности доменной логики.

## 2. Имплементация Локализации
### 2.1 Извлечение Ресурсов
Вместо зашитых строк типа `Text("Автозакрытие дня")`, используется нативная система локализации Android:
```kotlin
Text(stringResource(id = R.string.settings_auto_close_day))
```
Структура папок-ресурсов будет включать `values` (русский или английский как дефолт), `values-en`, `values-es`, `values-fr`, `values-de`.

### 2.2 Перехват языковых настроек в приложении
С выходом Android 13 Google внедрил систему независимой настройки языка приложения. Для обеспечения обратной совместимости, мы будем использовать библиотеку AndroidX:
```kotlin
val localeList = LocaleListCompat.forLanguageTags("en")
AppCompatDelegate.setApplicationLocales(localeList)
```

### 2.3 UI Выбора языка в Настройках
* Опция **"Язык по умолчанию" (System default)** добавляется первой (сохраняется как пустое значение `""`).
* За ней будет идти список доступных языков (например, в виде диалогового окна/выпадающего списка).
* При выборе нового языка, настройка синхронизируется с `DataStore`, и мгновенно вызывается `AppCompatDelegate` для перезагрузки строк.
