## ADDED Requirements

### Requirement: Таблица LampStatistics в Room
Система ДОЛЖНА хранить историю фактического пробега по каждой лампочке в таблице `lamp_statistics` с полями: `id` (PK), `profile_id` (FK), `lamp_index` (Int), `actual_distance` (Float, км), `timestamp` (Long).

#### Scenario: Создание таблицы при миграции
- **WHEN** приложение обновляется до версии с поддержкой LampStatistics
- **THEN** Room выполняет миграцию и создаёт таблицу `lamp_statistics`
- **AND** существующие данные в других таблицах НЕ затрагиваются

### Requirement: Скользящее окно записей
Система ДОЛЖНА поддерживать жёсткий лимит HISTORY_LIMIT = 5 записей для каждого `lamp_index` в рамках одного `profile_id`. При превышении лимита удаляется запись с минимальным `timestamp`.

#### Scenario: Вставка в пределах лимита
- **WHEN** для lamp_index=2 существует 3 записи
- **AND** записывается новый факт пробега
- **THEN** новая запись добавляется, общее количество записей = 4

#### Scenario: Вставка с превышением лимита
- **WHEN** для lamp_index=2 существует 5 записей (HISTORY_LIMIT)
- **AND** записывается новый факт пробега
- **THEN** запись с минимальным timestamp удаляется
- **AND** новая запись добавляется, общее количество записей = 5

### Requirement: Запись факта пробега при LED-событии
При фиксации LED-события система ДОЛЖНА вычислить дельту пробега (расстояние с момента предыдущего LED-события или старта) и сохранить в LampStatistics для соответствующего lamp_index.

#### Scenario: Фиксация первой лампочки за поездку
- **WHEN** пользователь фиксирует потухание лампочки (ledCount: 5→4)
- **AND** текущий пробег за день = 12.5 км, предыдущих LED-событий за день нет
- **THEN** система записывает LampStatistics: lamp_index=0, actual_distance=12.5

#### Scenario: Фиксация последующей лампочки
- **WHEN** пользователь фиксирует потухание лампочки (ledCount: 4→3)
- **AND** текущий пробег за день = 22.0 км, предыдущее LED-событие на 12.5 км
- **THEN** система записывает LampStatistics: lamp_index=1, actual_distance=9.5

### Requirement: Обработка пропусков фиксации
Если пользователь пропустил фиксацию одной или нескольких лампочек, система ДОЛЖНА распределить общую дельту пробега между пропущенными лампочками пропорционально их базовым весам.

#### Scenario: Пропущена одна лампочка
- **WHEN** пользователь фиксирует потухание лампочки (ledCount: 5→3, пропущена лампочка 4)
- **AND** текущий пробег = 25.0 км, предыдущих LED-событий нет
- **AND** профиль Type_5 с весами [0.30, 0.20, 0.20, 0.15, 0.15]
- **THEN** система распределяет 25 км между lamp_index=0 (вес 0.30) и lamp_index=1 (вес 0.20): actual_distance[0]=15.0, actual_distance[1]=10.0
- **AND** записывает два факта в LampStatistics

### Requirement: Расчёт прогноза по статистике (горячий старт)
Если для lamp_index имеются записи в LampStatistics, система ДОЛЖНА использовать среднее арифметическое поля actual_distance для прогноза. Если записей нет — использовать базовую формулу `maxMileage * weight[i]`.

#### Scenario: Прогноз с полной статистикой
- **WHEN** для lamp_index=0 существуют записи [12.0, 13.5, 11.0, 14.0, 12.5]
- **THEN** прогноз для lamp_index=0 = среднее(12.0, 13.5, 11.0, 14.0, 12.5) = 12.6 км

#### Scenario: Прогноз без статистики (холодный старт)
- **WHEN** для lamp_index=3 нет записей в LampStatistics
- **AND** maxMileage = 60 км, профиль Type_5, вес lamp_index=3 = 0.15
- **THEN** прогноз для lamp_index=3 = 60 * 0.15 = 9.0 км

### Requirement: Отображение статистики рядом с барабаном
В настройках батареи рядом с каждым барабаном лампочки система ДОЛЖНА показывать статистический пробег. Если отклонение статистики от установленного значения > 20% и статистика хуже — на экране «Старт» (Dashboard) показать предупреждение-ссылку.

#### Scenario: Статистика есть, отклонение в пределах нормы
- **WHEN** барабан лампочки 1 установлен на 18 км
- **AND** средняя статистика для лампочки 1 = 16 км (отклонение 11%)
- **THEN** рядом с барабаном отображается текст «Факт: ~16 км» серым цветом
- **AND** предупреждение НЕ показывается

#### Scenario: Статистика сильно хуже установленного
- **WHEN** барабан лампочки 1 установлен на 18 км
- **AND** средняя статистика для лампочки 1 = 12 км (отклонение 33%, хуже)
- **THEN** рядом с барабаном отображается текст «Факт: ~12 км» оранжевым цветом
- **AND** на экране «Старт» внизу появляется сообщение-ссылка: «Статистические данные сильно отличаются от установленных» → переход в раздел Батарея настроек

#### Scenario: Статистики нет
- **WHEN** для лампочки 4 нет записей в LampStatistics
- **THEN** рядом с барабаном отображается текст «нет данных» серым цветом
- **AND** предупреждение НЕ показывается
