## ADDED Requirements

### Requirement: Фиксированные профили весов батареи
Система ДОЛЖНА содержать неизменяемые массивы весов (долей ёмкости) для каждого типа пульта. Индексация: индекс 0 — самая верхняя (первая тухнущая) лампа, индекс N — нижняя (последняя).

Массивы весов:
- Type_3: [0.45, 0.35, 0.20]
- Type_4: [0.35, 0.30, 0.20, 0.15]
- Type_5: [0.30, 0.20, 0.20, 0.15, 0.15]
- Type_6: [0.25, 0.20, 0.15, 0.15, 0.15, 0.10]
- Type_8: [0.20, 0.15, 0.15, 0.12, 0.12, 0.12, 0.09, 0.05]
- Type_10: [0.13, 0.13, 0.13, 0.10, 0.10, 0.10, 0.10, 0.07, 0.07, 0.07]

#### Scenario: Получение весов для Type_5
- **WHEN** запрашиваются веса для ledCount = 5
- **THEN** возвращается массив [0.30, 0.20, 0.20, 0.15, 0.15]
- **AND** сумма элементов массива = 1.0

#### Scenario: Fallback для неизвестного ledCount
- **WHEN** запрашиваются веса для ledCount = 7 (не входит в фиксированный набор)
- **THEN** возвращается массив из 7 равных долей (1/7 каждая)

### Requirement: Фиксированный выбор количества лампочек
В настройках батареи количество лампочек ДОЛЖНО выбираться из фиксированного набора: 3, 4, 5, 6, 8, 10.

#### Scenario: Выбор количества лампочек через dropdown
- **WHEN** пользователь открывает настройки батареи
- **THEN** вместо WheelNumberPicker (range 1..10) отображается dropdown с вариантами: 3, 4, 5, 6, 8, 10

#### Scenario: Изменение ledCount пересчитывает ledDistances
- **WHEN** пользователь меняет ledCount с 5 на 6
- **AND** текущий maxMileage = 60 км
- **THEN** ledDistances пересчитывается по весам Type_6: [15.0, 12.0, 9.0, 9.0, 9.0, 6.0]

### Requirement: Порядок элементов в разделе Батарея
В настройках раздел «Батарея» ДОЛЖЕН содержать элементы в следующем порядке: 1) Лампочки на пульте (dropdown), 2) Пробег на одном заряде (WheelPicker общий), 3) Индивидуальные барабаны для каждой лампочки.

#### Scenario: Порядок отображения
- **WHEN** пользователь открывает настройки батареи
- **THEN** первым элементом идёт «Лампочки на пульте» (dropdown)
- **AND** вторым — «Пробег на одном заряде» (WheelPicker)
- **AND** далее — индивидуальные барабаны для каждой лампочки

### Requirement: Индивидуальные барабаны для каждой лампочки
Для каждой лампочки из ledCount ДОЛЖЕН отображаться собственный WheelNumberPicker с пробегом в км.

#### Scenario: Изменение общего пробега пересчитывает лампочки
- **WHEN** пользователь изменяет общий пробег с 60 на 80 км
- **AND** профиль Type_5
- **THEN** барабаны лампочек пересчитываются: [24.0, 16.0, 16.0, 12.0, 12.0]

#### Scenario: Изменение барабана лампочки пересчитывает общий
- **WHEN** пользователь изменяет барабан лампочки 1 с 18 на 20 км
- **AND** остальные лампочки: [12, 12, 9, 9]
- **THEN** общий пробег обновляется: 20 + 12 + 12 + 9 + 9 = 62 км

### Requirement: Адаптивное отображение LED-индикаторов на Dashboard
Dashboard ДОЛЖЕН корректно отображать 3, 4, 5, 6, 8 и 10 LED-индикаторов, адаптируя размер и spacing.

#### Scenario: Отображение 5 лампочек (стандарт)
- **WHEN** у профиля ledCount = 5
- **THEN** индикаторы отображаются размером 40dp (активные) / 32dp (неактивные) с spacing 12dp

#### Scenario: Отображение 8 лампочек
- **WHEN** у профиля ledCount = 8
- **THEN** индикаторы отображаются размером 28dp / 22dp с spacing 6dp
- **AND** все 8 помещаются в одну строку

#### Scenario: Отображение 10 лампочек
- **WHEN** у профиля ledCount = 10
- **THEN** индикаторы отображаются размером 22dp / 18dp с spacing 4dp
- **AND** все 10 помещаются в одну строку

### Requirement: Динамическая цветовая шкала LED
Цвета LED ДОЛЖНЫ генерироваться динамически для любого ledCount от 3 до 10 по градиенту: красный (нижняя) → оранжевый (средние) → зелёный (верхние).

#### Scenario: Генерация цветов для 6 лампочек
- **WHEN** ledCount = 6
- **THEN** цветовая шкала содержит 6 цветов от красного (index=1) до зелёного (index=6)

### Requirement: Удаление кнопки «Настроить голосовые команды»
Из раздела «Голосовые команды» ДОЛЖНА быть удалена кнопка `OutlinedButton` «Настроить голосовые команды». Параметры вибрации и звука остаются.

#### Scenario: Раздел голосовых команд без кнопки настройки
- **WHEN** пользователь открывает настройки
- **THEN** в разделе «Голосовые команды» отображаются только переключатели вибрации и звука
- **AND** кнопка «Настроить голосовые команды» отсутствует

### Requirement: Ссылка «Обратиться к разработчику» в настройках
Рядом с версией приложения ДОЛЖНА отображаться локализованная ссылка для связи по email.

#### Scenario: Отображение ссылки
- **WHEN** пользователь прокручивает настройки до конца
- **THEN** рядом с версией приложения отображается текст «Обратиться к разработчику по email» как кликабельная ссылка
- **AND** текст локализован на язык, выбранный в системе

### Requirement: Исправление расчёта Точки Невозврата
Точка невозврата ДОЛЖНА рассчитываться как `maxMileage / 2.0` (фиксированное значение из паспорта профиля), а не на основе динамического forecast.

#### Scenario: PoNR при 60 км общего пробега
- **WHEN** maxMileage профиля = 60 км
- **AND** ponrWarningKm = 2
- **THEN** PoNR = 30 км
- **AND** предупреждение озвучивается при пробеге >= 28 км (30 - 2)

#### Scenario: PoNR НЕ срабатывает ложно на малых дистанциях
- **WHEN** maxMileage профиля = 60 км
- **AND** текущий пробег = 3 км
- **THEN** предупреждение НЕ озвучивается
