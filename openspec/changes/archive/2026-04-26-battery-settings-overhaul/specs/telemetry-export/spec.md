## ADDED Requirements

### Requirement: Категории экспортируемых данных
Система ДОЛЖНА разделять экспортируемые данные на 6 категорий с возможностью выбора пользователем.

Категории:
1. **Информация об устройстве** (🔒 всегда включена) — модель, версия Android, версия приложения, RAM, размер экрана, уровень батареи телефона
2. **Настройки профиля** (🔒 всегда включена) — полный снапшот: maxMileage, ledCount, ledDistances, maxSpeed, type, TTS-режим/интервал, PoNR-порог, единицы измерения, auto-transport, use-statistics, язык
3. **Данные батареи и лампочек** (по умолчанию ВКЛ) — LedEvent-ы, LampStatistics, текущий forecast, текущий LED count
4. **История поездок** (по умолчанию ВКЛ) — DaySession (даты, total_distance), TrackSegment (start/end time, distance) — без координат
5. **Журнал ошибок** (по умолчанию ВКЛ) — перехваченные исключения, stack traces, контекст ошибки
6. **GPS-координаты** (по умолчанию ВЫКЛ) — LocationPoint (lat, lon, timestamp, isTransport) — большой объём

#### Scenario: Пользователь видит все категории
- **WHEN** пользователь инициирует экспорт данных
- **THEN** отображается диалог с 6 чекбоксами, где категории 1 и 2 заблокированы (всегда включены), категории 3-5 включены, категория 6 выключена

#### Scenario: Пользователь отключает категорию
- **WHEN** пользователь снимает чекбокс «История поездок»
- **THEN** в ZIP-архив НЕ включаются файлы sessions.json и tracks.json

### Requirement: Полный снапшот настроек в экспорте
Файл `settings.json` ДОЛЖЕН содержать полный снапшот всех настроек приложения, включая все параметры активного профиля и глобальные настройки.

#### Scenario: Содержание settings.json
- **WHEN** формируется ZIP-архив
- **THEN** файл `settings.json` содержит: profile (id, name, type, maxSpeed, maxMileage, ledCount, ledDistances), tts (mode, distanceIntervalKm, timeIntervalMin), ponr (warningKm), general (unitSystem, autoTransportDetection, useStatistics, appLanguage)

### Requirement: Оценка размера архива
Диалог экспорта ДОЛЖЕН показывать оценочный размер итогового архива, пересчитываемый при включении/выключении чекбоксов.

#### Scenario: Пересчёт размера при включении GPS
- **WHEN** пользователь включает чекбокс «GPS-координаты»
- **THEN** оценочный размер архива увеличивается на примерный объём LocationPoint данных

### Requirement: Сбор ошибок в локальный файл
Система ДОЛЖНА перехватывать необработанные исключения и записывать в файл `filesDir/telemetry/errors.jsonl` в формате JSON Lines с полями: timestamp, errorType, message, stackTrace.

#### Scenario: Запись необработанного исключения
- **WHEN** в приложении происходит необработанное исключение
- **THEN** система записывает строку JSON с полями timestamp, errorType, message, stackTrace в файл errors.jsonl
- **AND** приложение продолжает стандартную обработку крэша

### Requirement: Формирование ZIP-архива по выбранным категориям
Система ДОЛЖНА формировать ZIP-архив, включая только файлы выбранных пользователем категорий. Данные ограничиваются последними 30 днями.

#### Scenario: Экспорт без GPS-координат
- **WHEN** пользователь подтверждает экспорт с выключенным чекбоксом GPS
- **THEN** ZIP содержит: device_info.json, settings.json, led_events.json, lamp_statistics.json, sessions.json, tracks.json, errors.jsonl
- **AND** ZIP НЕ содержит locations.json

#### Scenario: Экспорт с GPS-координатами
- **WHEN** пользователь включает чекбокс GPS и подтверждает экспорт
- **THEN** ZIP дополнительно содержит locations.json

### Requirement: Отправка по email через Intent
По нажатию кнопки «Отправить» система ДОЛЖНА открыть почтовый клиент через Intent.ACTION_SEND с получателем sumaneev@gmail.com и ZIP-вложением.

#### Scenario: Успешная отправка
- **WHEN** пользователь нажимает «Отправить» в диалоге экспорта
- **THEN** система формирует ZIP-архив
- **AND** открывает почтовый клиент с адресом sumaneev@gmail.com, темой «FigaGo Telemetry v{versionName}» и ZIP-вложением

### Requirement: Ссылка «Обратиться к разработчику» в настройках
Рядом с версией приложения ДОЛЖНА отображаться локализованная ссылка для связи по email. При нажатии — открывается диалог экспорта с чекбоксами.

#### Scenario: Отображение ссылки
- **WHEN** пользователь прокручивает настройки до конца
- **THEN** под версией приложения отображается кликабельный текст «Обратиться к разработчику по email»
- **AND** текст локализован на язык, выбранный в системе
