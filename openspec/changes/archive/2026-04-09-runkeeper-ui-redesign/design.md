## Context

Доработка `init-figago` реализовала базовую функциональность записи трека, где все элементы (карта, кнопки начала/конца дня, лампочки) были просто сложены на один Главный экран. Для комфортного управления приложением на ходу или в коляске требуется переработать навигацию, внедрив `BottomNavigationBar` и выделив огромный "Дашборд-экран" для активного режима записи трека (аналог UI из Runkeeper). 
Кроме того, иногда пользователь пересаживается в автомобиль বা автобус, забыв выключить запись, что портит статистику для дальнейшего прогноза батареи. Требуется детектор превышения максимальной скорости коляски.

## Goals / Non-Goals

**Goals:**
- Внедрить `Scaffold` с `BottomNavigationBar` (Я, История, Старт, Настройки) как основной каркас приложения.
- Разделить состояния Главного экрана: до старта — Карта пользователя, после старта — Дашборд с огромными цифрами.
- На Дашборде использовать визуальные иконки для управления треком (Пауза `||`, Старт `▶️`, Стоп `⏹`) вместо графического текста.
- Реализовать автоматическую паузу в `TrackingService`, если скорость преодолевает заданный порог (превышение скорости).

**Non-Goals:**
- Мы не меняем алгоритмы прогнозирования (`ForecastRemainingDistanceUseCase`) или работу с БД — они остаются прежними, меняется только их UI-обертка.

## Decisions

### 1. Переработка Навигации (Bottom Navigation)
Вместо одного сложного экрана `MainScreen` мы переходим к стандартной компоновке с `NavHost` внутри `Scaffold`. 
- **Элементы панели:** Map ("Я"), History ("История"), Запуск/Дашборд ("Старт"), Settings ("Настройки").
- Кнопка "Старт" в меню навигации может физически запускать новую сессию/отрезок, если он не запущен (через вызов во ViewModel), и переключать фокус на вкладку Дашборда.

### 2. Структура Дашборда (Dashboard Screen)
Экран дашборда активного трека будет построен на `Column` с центрированием:
- `displayLarge` для таймера (`00:15`).
- Индикаторы лампочек (InteractiveLedIndicators из прошлых задач).
- Вложенные `Text` для пройденных километров и прогнозных.
- По центру внизу: Круглый `FloatingActionButton` (FAB) для Паузы (`||`). Когда стейт = `PAUSED`, вместо одной кнопки показывается `Row` с двумя кнопками (Зеленая `Play` и Красная `Stop`).

### 3. Авто-Пауза при превышении скорости (Speed-based Auto-Pause)
Максимальная возможная скорость сильно варьируется в зависимости от модели инвалидной коляски (от 6-7 км/ч до 15 км/ч и выше). Пользователь может пересаживаться в транспорт, забыв нажать стоп.
- **Реализация:** В `TrackingService.kt` при приеме `location` алгоритм проверяет поле `location.speed` (м/с -> км/ч). Если текущая скорость превышает заданный лимит (`maxSpeedLimit`), трек ставится на паузу (`DayState.PAUSED`), без закрытия дневной сессии.
- В `SettingsRepository` добавим ползунок/настройку "Лимит скорости для Авто-паузы, км/ч" (с возможностью задать низкие значения, например 6-7 км/ч).

## Risks / Trade-offs

- **Риск:** Пользователь по ошибке нажмет кнопку "Стоп" (`⏹`) вместо "Старт" (`▶️`) на экране паузы, потому что они просто иконки.
  - **Решение:** Делаем их контрастными. Старт/Возобновить — зеленым цветом, Стоп — красным с иконкой квадрата.
- **Риск:** Скачки GPS могут выдать случайную скорость в 50 км/ч на долю секунды, сработает ложная Авто-пауза.
  - **Решение:** Авто-пауза должна срабатывать только в том случае, если превышение зафиксировано строго в 3-х последовательных точках `Location`, отсекая случайные погрешности GPS.

### 4. Интеграция в системный UI (Шторка и Уведомления)
Системный интерфейс Android будет активно использоваться для быстрого доступа к приложению и управления им:
- **Foreground Service Notification (Вариант А):** Уведомление `TrackingService` (которое висит в шторке во время записи) будет расширено кнопками действий (`NotificationCompat.Action`). В него будут добавлены иконки "Стоп", "Пауза" и "Смена лампочки", чтобы управлять поездкой без открытия приложения.
- **Quick Settings Tile (Вариант Б):** Будет реализован `TileService`, который добавит кастомную кнопку в панель быстрых настроек Android. Иконкой послужит отдельный ассет, а клик по тайлу будет просто запускать/открывать приложение `FigaGo` (Dashboard) для быстрого старта поездки.
