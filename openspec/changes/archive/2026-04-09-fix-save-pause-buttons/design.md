## Context

Приложение позволяет записывать GPS-треки. Во время работы пользователи столкнулись с проблемами в пользовательском интерфейсе: кнопка "Сохранить" (вероятно, в настройках или при завершении) не реагирует на нажатия, а кнопка "Пауза" после запуска трека остается неактивной, не позволяя приостановить запись. 

## Goals / Non-Goals

**Goals:**
- Исправить работу кнопки "Сохранить", чтобы она корректно триггерила сохранение данных (передавала Intent/Action в ViewModel).
- Исправить логику активации кнопки "Пауза" в момент начала записи трека.
- Убедиться, что стейт UI соответствует реальному состоянию Foreground Service или ViewModel.

**Non-Goals:**
- Полный редизайн экранов или изменение способа записи.
- Добавление новой функциональности, не связанной с кнопками сохранения и паузы.

## Decisions

1. **State Management UI Fix**: Проблема с кнопкой паузы, скорее всего, связана с тем, что состояние UI (например, `RecordingState`) не обновляется или кнопке не передается правильное значение `enabled`. Будет проведена ревизия потоков состояния (`StateFlow`), питающих `MainScreen` / `SettingsScreen`.
2. **Event Dispatching Fix**: Проблема с кнопкой сохранения, скорее всего, связана с отсутствием вызова метода ViewModel в `onClick` или ошибкой в валидации формы (если она есть), которая блокирует кнопку.

## Risks / Trade-offs

- Риск поломки других состояний кнопок при обновлении стейт-машины → Требуется ручное тестирование переходов Старт -> Пауза -> Старт -> Финиш.
