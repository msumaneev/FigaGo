## ADDED Requirements

### Requirement: Detect Transport Automatically
Система должна уметь распознавать переход пользователя в транспорт с помощью системного API (ActivityRecognition).

#### Scenario: Switching transit modes without fast movement
- **WHEN** GPS сообщает скорость 5 км/ч, но `ActivityRecognitionClient` отдает состояние `IN_VEHICLE` с высокой вероятностью
- **THEN** Приложение должно установить `isTransportMode = true` и отметить текущие записываемые сегменты как "Транспорт" для корректной маршрутизации.

### Requirement: Ignore Cold GPS Jumps
Для исключения намотки несуществующей дистанции при старте трека, приложение должно игнорировать мусорные точки («звезду»).

#### Scenario: Location jump right at the beginning
- **WHEN** получена GPS-точка с погрешностью (`accuracy`) > 30 метров в течение первых 15-30 секунд
- **THEN** Приложение должно не засчитывать пройденную дистанцию для этой точки и ожидать стабилизации сигнала.

### Requirement: Manage Sleep/Awake gaps
Для исключения рисования прямой синей линии сквозь дома при убийстве фонового процесса.

#### Scenario: Android OS killed the service
- **WHEN** Приложение снова получает координаты, но разница во времени с последней точкой превышает 2 минуты
- **THEN** Алгоритм не должен рисовать прямую линию между ними, а должен принудительно начать новый под-сегмент (`TrackSegment`), визуально разрывая трек.
