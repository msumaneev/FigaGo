## ADDED Requirements

### Requirement: Pre-flight Permission Validation
Приложение SHALL проверять обязательные системные разрешения перед запуском фоновой геолокации.

#### Scenario: Missing background location permission
- **WHEN** пользователь нажимает "Старт" на главном экране при отсутствии прав на фоновую локацию
- **THEN** старт трекинга блокируется и открывается экран "Диагностика" с выделенной ошибкой

#### Scenario: Missing battery optimization exclusion
- **WHEN** пользователь нажимает "Старт" на главном экране при включенной агрессивной оптимизации батареи (не в исключениях)
- **THEN** старт трекинга блокируется и открывается экран "Диагностика" с выделенной ошибкой

### Requirement: Diagnostic Screen Actions
Экран "Диагностика" SHALL предоставлять визуальную индикацию статуса и прямые переходы в системные настройки для исправления проблем.

#### Scenario: Open diagnostics
- **WHEN** пользователь открывает экран "Диагностика"
- **THEN** система отображает список проверок, показывая ✅ для пройденных и ⚠️ с кнопкой "Исправить" для отсутствующих

#### Scenario: Fixing battery optimization
- **WHEN** пользователь нажимает кнопку "Исправить" рядом с ошибкой энергосбережения
- **THEN** система вызывает интент `Settings.ACTION_REQUEST_IGNORE_BATTERY_OPTIMIZATIONS`, а при отказе / отсутствии поддержки переводит на детали приложения

#### Scenario: Fixing location provider
- **WHEN** пользователь нажимает "Исправить" рядом с отключенным GPS-модулем
- **THEN** система открывает `Settings.ACTION_LOCATION_SOURCE_SETTINGS`
