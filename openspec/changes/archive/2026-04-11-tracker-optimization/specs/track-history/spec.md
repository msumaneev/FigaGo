## ADDED Requirements

### Requirement: Profile Isolated History
Система SHALL привязывать новые поездки к `profileId` активной коляски и фильтровать вывод списка соответственно.

#### Scenario: Saving new track
- **WHEN** сохраняется новая `DaySessionEntity` поездки
- **THEN** в таблицу базы данных записывается идентификатор текущего активного профиля

#### Scenario: Viewing filtered history
- **WHEN** пользователь открывает вкладку "История"
- **THEN** система отображает только те поездки, `profileId` которых равен идентификатору выбранного в данный момент профиля (в карусели)

### Requirement: No Nested Navigation UI
Экран истории SHALL быть оформлен как корневая вкладка без дублирующей навигации.

#### Scenario: Navigating to history
- **WHEN** пользователь переходит на вкладку "История" через нижнее меню
- **THEN** на экране отсутствует верхняя панель (`TopAppBar`), нет заголовка "История" и нет стрелки возврата назад
